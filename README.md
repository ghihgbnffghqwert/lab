# Лабораторна робота 5 — Документування коду

Односторінковий лендінг (послуги, новини, контактна форма, чат) на PHP без фреймворку.

## Вимоги

- PHP 7.4+
- MySQL/MariaDB
- Composer (для генерації документації)

## Швидкий старт

1. Налаштуйте БД: створіть базу, виконайте SQL з `schema.sql` (якщо є).
2. Скопіюйте `.env.example` у `.env` і вкажіть `DB_HOST`, `DB_USER`, `DB_PASS`, `DB_NAME` (або змініть значення за замовчуванням у `db.php`).
3. За потреби налаштуйте веб-сервер (Apache/Nginx) так, щоб корінь сайту вказував на каталог проєкту.
4. Відкрийте головну сторінку в браузері.

## Генерація документації

Див. **[docs/generate_docs.md](docs/generate_docs.md)** — повна інструкція.

Коротко:

```bash
composer install
composer run-script docs
```

Відкрийте `docs/phpdoc/index.html` у браузері.

## Що і як документувати (для тих, хто долучається до проєкту)

Щоб усі підтримували однаковий рівень документації:

### 1. Стандарт коментарів — PHPDoc

- Кожен **файл** з логікою має на початку блок `/** ... */` з коротким описом призначення файлу.
- Кожна **функція** і **метод** мають PHPDoc з:
  - одним або кількома реченнями опису;
  - `@param тип $ім'я опис` для кожного аргументу;
  - `@return тип опис` для функцій/методів, що щось повертають;
  - за потреби: `@throws`, `@see`, `@example`.

Приклад:

```php
/**
 * Екранує рядок для безпечного виводу в HTML.
 *
 * @param string $s Вхідний рядок
 * @return string Екранований рядок
 */
function h($s) {
    return htmlspecialchars((string) $s, ENT_QUOTES, 'UTF-8');
}
```

### 2. Що саме документувати

- **Публічні інтерфейси:** усі функції та методи, доступні з інших файлів.
- **Архітектура та рішення:** зміни в структурі проєкту або важливі рішення описувати в `docs/ARCHITECTURE.md`.
- **Складні алгоритми:** коротко в PHPDoc або в окремому пункті у `docs/`.
- **Взаємодія компонентів:** який файл що викликає — у файловому PHPDoc через `@see`.

### 3. Підтримка актуальності

- Документація — не одноразова акція. При зміні сигнатури функції, параметрів або логіки оновлюйте відповідні коментарі та `docs/ARCHITECTURE.md`.
- Після змін у коді варто перегенерувати документацію: `composer run-script docs`.

### 4. Де що лежить

| Що | Де |
|----|-----|
| Інструкція з генерації | `docs/generate_docs.md` |
| Архітектура та взаємодія компонентів | `docs/ARCHITECTURE.md` |
