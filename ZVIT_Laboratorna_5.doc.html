Тема: Документування коду

Мета роботи: ознайомитися із сучасними підходами та інструментами документування програмного коду, оволодіти практичними навичками створення якісної документації для власного проєкту з урахуванням особливостей обраної мови програмування (PHP) та використаного підходу.

Що зроблено

  • Досліджено стандарти та рекомендації документування для PHP (PHPDoc); огляд інструментів автогенерації (phpDocumentor, Sami тощо).
  • Додано документацію до ключових елементів проєкту: index.php, db.php, lib/helpers.php (функція h()), actions/contact.php.
  • Оновлено README.md — опис того, що й як документувати, щоб усі, хто долучається до проєкту, дотримувалися єдиного підходу.
  • Встановлено та налаштовано інструмент автогенерації документації (phpDocumentor), створено інструкцію з генерації, згенеровано документацію та архів.
  • Документовано архітектуру та бізнес-логіку проєкту; створено специфікацію OpenAPI (Swagger) для ендпоінта контактної форми; налаштовано Swagger UI для інтерактивного відображення та перевірки прикладів.
  • Інтегровано генерацію документації з процесом збірки (composer-скрипти, phpdoc.xml); налаштовано CI/CD для автоматичної генерації документації (GitHub Actions).

  • Коміт з усією документацією: https://github.com/ghihgbnffghqwert/lab/commit/ed7bc0ad1fc3aff7fb99b2d5a996023faed652cc

2. Інструкція з генерації документації: docs/generate_docs.md

  • У ній описано: встановлення залежностей (composer install), команду генерації (composer run-script docs), відкриття результату (docs/phpdoc/index.html), очищення та інтеграцію з CI.

3. Згенерована документація окремим архівом

4. Інтеграція з процесом збірки: відповідні файли конфігурації та скрипти
