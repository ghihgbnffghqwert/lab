# Стандарти та інструменти документування коду (PHP)

## 1. Стандарти документування для PHP

### PHPDoc (PSR-5 / de facto стандарт)

- **Блоки коментарів** — перед кожним класом, методом, функцією коментар у форматі `/** ... */`.
- **Теги:**
  - `@param тип $name опис` — параметри функції/методу
  - `@return тип опис` — значення, що повертається
  - `@throws КласВинятку опис`
  - `@var тип` — тип властивості
  - `@package`, `@see`, `@example` — пакет, посилання, приклад
- **Типи:** можна використовувати union (`string|null`), array shapes тощо.
- **Мета:** покращення читабельності, автодоповнення в IDE, автоматична генерація документації.

### Рекомендації для проєкту

- Документувати усі публічні інтерфейси (функції, методи, класи).
- Для скриптів (наприклад `actions/contact.php`) — файловий PHPDoc з описом призначення та вхідних даних (POST).
- Підтримувати документацію при будь-яких змінах у коді (документація — не одноразова акція).

## 2. Огляд інструментів автогенерації документації

| Інструмент        | Призначення                    | Використання в проєкті   |
|-------------------|--------------------------------|---------------------------|
| **phpDocumentor** | Генерація HTML/PDF з PHPDoc    | Встановлено, використовується |
| **Sami**          | Генерація API-документації     | Альтернатива phpDocumentor    |
| **Psalm / PHPStan** | Статичний аналіз, перевірка типів | Можна використати для контролю якості |
| **phpcs + DocBlock sniffs** | Перевірка стилю коду та наявності docblock | Контроль якості документації |

У проєкті обрано **phpDocumentor 3**: генерація HTML-документації з коментарів PHPDoc, інтеграція з CI (скрипт у `composer.json`).
